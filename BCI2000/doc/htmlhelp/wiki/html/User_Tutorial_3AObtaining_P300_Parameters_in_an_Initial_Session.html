Obtaining P300 Parameters in an Initial Session

User Tutorial

Although the basic properties of the P300 evoked potential are the same for all individuals, the response's latency, width, and spatial pattern varies, and adaptation to individual parameters improves classification.

Thus, it is necessary to obtain these individual parameters prior to any classification experiments, in an initial session.

Experimental Design

In the initial session, the subject is presented with a P300 character matrix as pictured below. During each trial, the subject is asked to focus on one character in the matrix (i.e., the target character). Meanwhile, the rows and columns of the matrix will flash randomly and successively so that sometimes the flashing will correspond to the column or row containing the target character and sometimes it will not. Each of these events will elicit a corresponding brain response. The purpose of this initial session is to identify those features that discriminate between the desired and undesired rows/columns. Thus, the only difference between the initial (i.e., calibration) session and subsequent sessions is that in the latter the speller module will not suggest a target character. For this reason, the initial session is referred to as copy spelling while subsequent sessions are referred to as free spelling.

Performing the Initial Session

  • Start BCI2000 by double-clicking batch/P3Speller_<YourAmplifier>.bat. For repeated use, placing a link to this file on the desktop might be a good idea.
  • In the operator module, click "Config".
  • In the configuration window, click "Load Parameters", and load the system-specific parameter file saved previously at parms/fragments/<YourSystem>.prm.
  • Again, click "Load Parameters" to load parms/p3_tutorial/InitialP3Session.prm.
  • When you have a separate monitor for experimenter and subject, load the parameter file at parms/fragments/stdlib/DualMonitor.prm.
    • In your system's display properties configuration, make sure that the subject's monitor is configured to be located to the right of the main monitor.
    • Make sure the WindowLeft parameter matches the main monitor's actual pixel width.
    • You may need to adapt WindowTop, WindowWidth, and WindowHeight parameters as well; click "Set Config" to try the effect of your changes.
  • Under the "Storage" tab, enter the subject's initials into the SubjectName parameter.
  • Switch to the "Source" tab; for the ChannelNames parameter enter electrode locations corresponding to amplifier channels as a white-space separated list (e.g., Fz CPz Cz CP3 ...).
  • Click "Set Config".
  • Instruct the subject as described in the next section.
  • Click "Start".
  • The recording will finish automatically. If you want to acquire more data, just click "Resume" as often as you like.
  • Finally, click "Quit" to leave BCI2000.

Subject instruction

During the initial session, the subject's screen displays a rectangular matrix of letters. Letters are flashed in rows and columns.

  • You will see a matrix containing letters.
  • Above the matrix, there is a line of text displayed.
  • During the session, the text will be followed with a single letter, enclosed in parentheses. This is the "target letter". Your task is to count the number of flashes that occur for the target letter.
